CKV_AWS_356 · aws_iam_policy · aws_iam_policy_document
IAM policy uses Resource = "*" for KMS actions
Flags KMS permissions on Resource = "*" instead of a specific key ARN. AWS expects "*" for some legitimate calls, so KICS may report false positives here.
Critical
ℹKnown false positive (Checkov #5148, #5305): some versions still flag kms:GenerateDataKey or kms:Decrypt on Resource = "*" even with a Condition. kms:ListKeys legitimately needs "*" in many setups. If your security team agrees, document a skip with a short justification.

CKV_AWS_111 · aws_iam_policy_document
IAM policy allows write access without constraints
Calls out powerful KMS APIs such as kms:PutKeyPolicy or kms:DeleteKey on Resource = "*" with no Condition. kms:Decrypt can be a false positive; see the info box above.
High
⚠Known false positive (Checkov #5305): CKV_AWS_111 may fire on kms:Decrypt with a Condition block present. If the condition is valid, use a skip annotation with justification.
